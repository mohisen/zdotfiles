  • Dietrich

    Xfce DOES NOT depend on Gnome specifically. and uses GTk2/GTk3 which depend on the GLib Object System or GObject. GLib is the non-graphical routines abstracted out from GTk+. These underlying libraries are used by both the Gnome Project, Xfce and other projects.

    So, Xfce can continue to grow on its own separate tree with GTk3 as the framework going forward.

    This makes Xfce perhaps the best Desktop strategy next to KDE Plasma Workspace.

  • Kiki Novak

  • Tycho Softworks

    This is somewhat incorrect. There is for example already a port of the xfce4 theme engine to gtk3. That was considered the starting point, though it suffers from the ever changing api madness.

    I believe xfce 4.12 is going to support if you have xfce desktop components written for either gtk2 or 3, there is a lot of code cleanup being done for gtk3 migration, and some select components may (by 4.12) then be able to be compiled on either. The migration is slow, and will happen over multiple releases, but it already seems underway.

  • Kiki Novak

    Let's quote Xfce's main developer Nick Schermer in his last interview. "I’ve ported some parts of the desktop to GTK3 to give it some testing and I’m not a big fan of the results. Beside the fact that porting is a lot of work, the ported applications also used more memory and felt slower. On its own that’s not a huge big problem if GTK3 brings a lot of cool stuff for regular users, but it doesn’t.
    So I prefer we depend on GTK 2.24 in Xfce 4.12 and focus on cleaning and GSEAL to make the GTK3 switch less invasive."
